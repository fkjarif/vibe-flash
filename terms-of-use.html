Section 02

Description of Service

VibeFlash is a static, browser-based flashcard application. It allows users to create flashcard decks manually, import text from PDF files (processed locally via PDF.js), and optionally use AI models via the OpenRouter API to generate flashcards automatically.

Key technical characteristics of the Service:

  • No backend server. VibeFlash does not have a server-side component. There are no databases, user accounts, or authentication systems owned or operated by VibeFlash.
  • Client-side only. All flashcard data — including your decks, cards, and any stored API key — lives exclusively in your browser's IndexedDB local storage. It never leaves your device unless you explicitly trigger an AI generation request (see Section 3).
  • PDF processing is local. When you upload a PDF, all text extraction occurs in your browser using PDF.js. No file is transmitted to any VibeFlash server.
  • Statically hosted. The app is deployed as a static site on Netlify, meaning the files you receive are pre-built HTML, CSS, and JavaScript — there is no server processing your requests.

The Service is provided free of charge and may be modified, updated, or discontinued at any time without prior notice.

Section 03

Bring Your Own API Key (BYOK)

The AI-powered flashcard generation feature requires you to supply your own API key from OpenRouter (openrouter.ai). VibeFlash does not provide, proxy, or subsidize access to any AI model.

By using the BYOK feature, you acknowledge and agree to the following:

  • Key storage. Your API key is stored locally in your browser's IndexedDB. It is not transmitted to, seen by, or stored on any VibeFlash server.
  • Direct API calls. When you initiate AI generation, your browser makes a direct HTTP request to https://openrouter.ai/api/v1/. VibeFlash's servers are not involved in this transaction.
  • Your responsibility for costs. You are solely responsible for all API usage costs incurred under your OpenRouter account. VibeFlash has no visibility into your usage and will not reimburse any charges.
  • Your responsibility for security. You are solely responsible for keeping your API key confidential. Use a key with appropriate rate limits and credit caps. Revoke keys you no longer use at openrouter.ai/keys.
  • OpenRouter's terms apply. Your use of the OpenRouter API is also governed by OpenRouter's own Terms of Service.

Security Warning: Because VibeFlash is a client-side app, anyone with access to your device and browser DevTools (e.g., the Application → IndexedDB panel) can potentially read your stored API key. Never use a high-credit, unrestricted API key. Always use a key configured with spending limits. VibeFlash is not liable for unauthorized use of your API key.
